# Workflow: Slot & Dungeons TMA

> Правила совместной работы. KISS — только то, что нужно для слаженной работы.

---

## Цикл итерации

```
1. Ассистент предлагает план реализации итерации
   (ключевые шаги, структура файлов, отрезки кода)
   
2. Ожидание согласования от разработчика
   → Без согласования код НЕ пишется

3. Реализация согласованного плана
   → Только то, что в тасклисте текущей итерации

4. Инструкция по тестированию:
   - Что открыть
   - Что нажать
   - Что должно произойти

5. Ожидание подтверждения "работает"

6. Обновление tasklist.md:
   - Чекбоксы итерации → [x]
   - Таблица прогресса → ✅

7. Git commit:
   git commit -m "feat: итерация N — [краткое описание]"

8. Согласование перехода к следующей итерации
```

---

## Запреты

- ❌ **НЕ** переходить к следующей итерации без подтверждения "работает"
- ❌ **НЕ** рефакторить то, что не сломано
- ❌ **НЕ** добавлять фичи не из `tasklist.md` без обсуждения
- ❌ **НЕ** писать код до согласования плана итерации

---

## Формат плана итерации

Перед кодом ассистент предоставляет:

```
### Итерация N — [Название]

Что делаем:
- [шаг 1]
- [шаг 2]

Новые файлы:
- src/path/to/file.ts

Изменяемые файлы:
- src/path/to/existing.ts

Ключевой код:
[отрезок кода для согласования подхода]

Тест:
[что открыть, что нажать, что проверить]
```

## Git

```bash
# После завершения итерации
git add .
git commit -m "feat: итерация N — [описание]"
git push
```

Формат commit message:
- `feat:` — новая функциональность
- `fix:` — исправление
- `refactor:` — рефакторинг (только при необходимости)
- `docs:` — документация

---

## Обновление tasklist.md

После каждой итерации обновить:
1. Чекбоксы `[ ]` → `[x]`
2. Статус в таблице прогресса → `✅ Готово`
3. Следующая итерация → `🔄 В работе`

## Деплой на Vercel

```bash
# Первый деплой
npx vercel login          # один раз, авторизация через браузер
npx vercel --prod         # имя проекта: slot-daggers (только строчные, без спецсимволов)

# Обновление (повторный деплой)
npm run build             # проверить сборку локально
npx vercel --prod         # пересобрать и обновить URL
```

**Текущий Production URL:** https://slot-daggers.vercel.app

1. **Создать бота** → написать `@BotFather`:
   ```

   → имя: Slot Daggers Game
   → юзернейм: SlotDaggersBot (должен заканчиваться на Bot)
   → сохранить TOKEN
   ```

2. **Создать WebApp**:
   ```
   /newapp
   → выбрать бота
   → Title: Slot & Daggers
   → Short name: play
   → URL: https://slot-daggers.vercel.app
   ```

3. **Ссылка для игры:** `t.me/SlotDaggersBot/play`

> При каждом обновлении кода — только `npx vercel --prod`, ссылка не меняется.
